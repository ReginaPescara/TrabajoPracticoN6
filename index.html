  10. Con toda la investigación que generaron deben de generar un formulario que realice las siguientes funciones.
    1. ¿Qué obtenga los datos del input?.
    2. ¿Qué verifique los datos que se ingresan en el input?.
    3. ¿Qué el formulario presente una calculo de sumatoria para validar ¿Qué no sos un robot”?.

Respuestas

  1. En el mercado las versiones de JavaScript que hay son JavaScript 1.0, JavaScript 1.1, JavaScript 1.2, JavaScript 1.3, JavaScript 1.4, JavaScript 1.5, JavaScript 1.6, JavaScript 1.7, y luego están las versiones ECMAScript 5 y ECMAScript 6.
  2. Para integrar JavaScript a HTML hay 3 formas:
    • La primera manera es utilizando la etiqueta "Script", anidada a la etiqueta "Head" del cófigo HTML. Dentro de la miama estará todo el código JS.
    • La segunda también utiliza la etiqueta "Script" de la misma forma, con la diferencia de que estará anidada a la etiqueta "Body" e irá al final de todo el contenido de la página.
    • Y la tercera, la última, es utilizando un archivo externo con extensión “.js”, y enlazarlo a través de la etiqueta “Script” que llevará como atributo la ubicación del dicho archivo JS. Generalmente puede ir anidada en la etiqueta "Head", aunque también puede ser parte del "Body".

  3. JavaScript para las variables utiliza nombres que simbolizan su valor, por lo que no llevara un nombre cualquiera. Ya que JS diferencia entre mayúsculas y minúsculas, es indispensable saber que por ejemplo, las variables “a” y “A” contarán cada una como una variable. Para declarar una variable existen tres formas, con var para variables globales, con let para variables locales o con constant que le dará a la variable un valor constante que no se podrá modificar luego.

    Como tipos de datos existen 8:

    • Boolaneo: un valor verdadero o falso (true/false).
    • Null: una palabra que determina un valor nulo.
    • Undifined: cuando un dato no esté definido.
    • Number: un número, ya sea entero o con coma.
    • BigInit: un número del tipo entero que tendrá precisión arbitraria.
    • String: un valor de texto conformado por una secuencia de caracteres.
    • Symbol: devuelve una propiedad única para un objeto.
    • Object: serán datos definidos.
    Todos estos son los tipos de datos, que JS no necesita que se declaren en las variables ya que es un lenguaje de programación dinámico. Lo que por ejemplo nos permite asignarle un número a una variable y después que sea posible asignarle un texto sin ningún tipo de problema.

    Y las expresiones regulares en JS se utilizan para poder extraer cierta parte de un conjunto de datos que cumplan con los requisitos pedidos, es decir, buscar aquellos que tengan la restricción y así poder extraerlos, modificarlo, etc. Todo esto es gracias al uso de patrones de cadenas de texto, donde algunos símbolos representan significados especiales.

  4. Dentro de todas las formas de declaraciones de funciones en JS están:
    • Function declaration (declaración de función): la estructura es básica, se crea con la palabra function seguida del nombre que le queramos dar para identificar, seguido de una lista de parámetros entre paréntesis y por último irán las llaves donde estarán todas las sentencias. Y para que este parámetro devuelva un valor, se utiliza la instrucción return.
    • Function expression (declaración de expresión): con una estructura similar a la forma anterior, con la diferencia de que es opcional usar la palabra function y asignarle un nombre. Además, este tipo de función permite almacenarla en una variable y eso permite reducir código.
    • IIFE (Immediately Invoked Function Expression): esta forma sirve para cuando queremos hacer uso una sola vez de una función. La estructura básica estará anidada a un par de paréntesis y al final irá otro par a diferencia de las otras. Y todas las variables que se vayan a utilizar en dicha función, tendrán que estar definidas dentro de los paréntesis.
    • Shorthand method definition: puede usarse como método en la declaración de un objeto o en las clases, con estructura básica.
    • Arrow function: de esta forma se declaran la lista de parámetros entre paréntesis, seguido de los símbolos => y con las llaves para las instrucciones. No siempre es necesario el uso de las llaves o de la instrucción return.
    • Generator function: con una estructura básica, esta función nos favorece cuando queremos que en un punto de las sentencias se devuelva algún valor y después se siga con el resto de instrucciones de la función. También utiliza la estructura básica, solo se le agrega un asterisco antes del nombre y que retorna un objeto.
    • Function Constructor: de esta forma podremos crear una función a medida que el programa se vaya ejecutando. Es decir, la estructura empezará declarando una variable como un objeto de function, y dentro del parámetro se encontrarán las variables que utilizaremos junto con el argumento al final, que esperamos que realice la función. Este tipo de función es muy poco utilizada.

    5. Los operadores se dividen en 10 tipos, estos son:

    • Operadores de asignación (por ejemplo x = 19).
    • Operadores de comparación (por ejemplo num == 10).
    • Operadores aritméticos (por ejemplo 25 % 5).
    • Operadores bit a bit (por ejemplo con el número binario 011 trabajaría, y devolvería 3).
    • Operadores lógicos (por ejemplo cont==0 && num==5).
    • Operadores de cadena (por ejemplo num2 += 1).
    • Operadores condicional (por ejemplo var (altura>160) ? ”altura promedio” : ”Altura por debajo del promedio”;).
    • Operador coma (por ejemplo var b = (3, 5, 4)).
    • Operadores unarios (por ejemplo delete name.edad).
    • Operadores relacionales (por ejemplo 7 in names).


    Y la conversión entre tipos ocurre cuando los operadores y funciones automáticamente son convertidos al tipo correcto de valor. Estos tipos son ToString, para pasar cualquier dato a representarlo de forma de texto. ToNumber, para pasar cualquier dato en funciones matemáticas y expresiones a una representación numérica. ToBoolean, pasar datos en operaciones lógicas a una representación con valor de verdad o falso.

    Las estructuras de control en JS son If simple, esta estructura nos permite realizar una sentencia si es que la variable con la que trabajemos cumple una condición. If/Else, esta estructura es igual a la de If simple, con la diferencia de que si la variable no cumple la condición, pasará a cumplirse otra sentencia. Ambas estructuras se pueden anidar a otra estructura similar, formando un múltiplo Else.


    Después está la estructura del Switch, según un valor dado esta estructura de control hará la comparación de este con una serie de casos, y si el valor coincide con alguna se realizará la sentencia de dicho caso. En caso contrario, habrá una sentencia que nos indique que el valor no cumple con ninguno de los casos. Bucle For esta estructura permite repetir las sentencias del código una cierta cantidad de veces, tiene una condicion al principio de la estructura que se asegurará de que el contador cumpla con la cantidad de vueltas asignada.


    También esta While que con esta estructura también podremos repetir sentencias de código, con una condición al principio que corroboran que una variable, generalmente un contador, cumpla y que se repetirá hasta que dicha variable no cumpla. De modo similar trabaja el bucle Do/While, con la diferencia de que la condición estará al final de la estructura, y por lo menos se ejecutarán las sentencias aunque sea 1 vez. Después si sigue o no depende de que la variable cumpla la condición.
